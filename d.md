1. Пріоритетне завантаження шрифтів (Font Preloading):



Перевір файл app/layout.tsx. Нам потрібно розірвати ланцюжок "CSS -> Font".

Якщо ми використовуємо next/font/local або next/font/google, переконайся, що вони підключені з параметром display: 'swap' та preload: true.

Додай у head (через metadata або компонент <link>) прямий прелоад для основного шрифту:<link rel="preload" href="/fonts/your-font.woff2" as="font" type="font/woff2" crossorigin="anonymous" />. Це змусить браузер почати завантаження шрифту одночасно з HTML.

2. Оптимізація головного зображення (Hero Image):



Звіт PageSpeed вказує, що hero.webp все ще можна стиснути.

У компоненті next/image для Hero:

Встанови quality={75} (або 70).

Перевір sizes. Якщо на мобільних це зображення займає весь екран, має бути sizes="100vw".

Додай атрибут fetchPriority="high" (саме з великою "P", якщо це властивість пропса у Next.js).

3. Мінімізація "Render-Blocking" ресурсів:



Проаналізуй файли стилів. Якщо у нас є великі бібліотеки (наприклад, для каруселей або календарів), які не потрібні миттєво, винеси їх у динамічні імпорти.

Для сторонніх скриптів (Google Analytics, Facebook Pixel тощо) у layout.tsx використай стратегію strategy="afterInteractive" або lazyOnload.

4. Усунення CLS та Layout Shift:



PageSpeed зафіксував невеликий зсув (0.048). Перевір, чи є у верхній частині сторінки (header, promo-bar) елементи без фіксованої висоти. Додай min-height для контейнера навігації, щоб він не змінював розмір після завантаження стилів.

5. Видалення поліфілів та зайвого коду:



Звіт знову вказує на Array.prototype.at та Object.hasOwn.

Перевір tsconfig.json і переконайся, що target стоїть як мінімум ES2020.

У next.config.js спробуй увімкнути експериментальну опцію optimizePackageImports, щоб Cursor міг оптимізувати імпорти з великих бібліотек.
